# Моделирование динамического аукциона с бюджетными ограничениями

В этом проекте реализована симуляция аукциона, которая может быть полезна для систем рекомендаций и других сценариев, где необходимо предсказать поведение пользователей в условиях ограниченных ресурсов. 

## Основные характеристики

1. **Динамическая стратегия ставок**: Пользователи делают ставки на основе предпочтений к предметам, которые выражаются в ставках. Эти ставки динамически корректируются с учетом уменьшения бюджета и приближения к финальным раундам, что помогает моделировать более осторожное поведение пользователей. Ставки регулируются функцией `update_bid_strategy`, которая снижает ставки при минимальном сокращении бюджета и приближении к последним раундам.

2. **Ограничения бюджета**: Ставки, превышающие доступный бюджет пользователя, обнуляются. Это ограничение позволяет пользователям делать ставки только в пределах оставшихся средств, что помогает контролировать баланс бюджета для каждого участника на протяжении всех раундов.

3. **Результаты и визуализация**: Код сохраняет историю выигрышей и доходов по каждому раунду и предоставляет графическую визуализацию с использованием библиотеки Seaborn. Визуализация включает графики:
   - Выигравшие ставки по каждому предмету для всех раундов, что позволяет анализировать результаты по каждому раунду аукциона.
   - Использование бюджета каждым пользователем, что позволяет оценить, насколько рационально и эффективно они расходуют средства.

4. **Возможное развитие**: Проект может быть дополнен алгоритмами машинного обучения, такими как совместная фильтрация, для прогнозирования предпочтений пользователей. Это добавление может повысить точность прогнозов поведения участников аукциона и распределения предметов, что особенно актуально для рекомендательных систем.

## Структура кода

- **Импорт библиотек**: Используются библиотеки NumPy, Pandas и Seaborn для численных расчетов, обработки данных и визуализации.
- **Определение параметров**: Определяются параметры аукциона — количество пользователей, предметов, раундов, а также изначальные бюджеты пользователей и предпочтения к предметам.
- **Функция корректировки ставок `update_bid_strategy`**: Динамически регулирует ставки с учетом текущего бюджета и оставшихся раундов.
- **Основная функция `run_auction`**: Проводит аукцион, распределяя предметы и контролируя бюджет пользователей в каждом раунде.
- **Визуализация результатов**: Использует библиотеку Seaborn для визуализации выигрышей и расхода бюджета пользователями, что позволяет получить полезную аналитику по результатам аукциона.

Этот проект представляет собой простую, но гибкую основу для моделирования поведения пользователей в аукционе и может быть полезен как для рекомендательных систем, так и для других приложений, где важны **ограничения ресурсов** и **конкурентные ставки**.

Для изучения дополнительных возможностей библиотеки Seaborn по визуализации см. [официальную документацию](https://seaborn.pydata.org/).
